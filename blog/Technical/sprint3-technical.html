    I like to think of HTML like the basic bones of a project. For example if you want to you could think of html like it is a bicycle. You can make a bike be very simple - the frame, two wheels, a basic paint job, a system for brakes and for changing gears, and basic handlebars. Lets say this bike is your one for all terrains type bike. So decent wide tyres to grip everything. And thats it. This bike is very basic and will get you from A to B.

    Then you can use CSS, which is the way of styling your bike/html. With CSS, you can modify everything to look very, very fancy. You can add lights for safety, reflectors, modify the frame and the tyres for lightness and speed or durability and extra grip.

    I would explain what control flow and loops were but I may have died at the point that it came up.

    The DOM, document/object/model, is a very fun tool you can use to mess with a website locally to your computer, or help you to modify and fix something with a webpage you are making. Recently I *struggled* through a DOM youtube crashcourse. The way to access it is through the webpage you are currently on, rightclicking and selecting inspect. Or however else you can access a inspector.
    Using it you can see which tag in your HTML script is causing a problem, for example: a margin or box that is forcing itself too far to the left, or something that you really dont like on the page. Using the inspector you can find out which line is causing the issue, and then change the code to fix the problem. And it is cool because you can deactivate element attribues or, modify them to see what will happen to the webpage - Live!!!
